## NOAA HRRR — 3 km, and virtual

[NOAA's High-Resolution Rapid Refresh](https://dynamical.org/catalog/noaa-hrrr-forecast-48-hour-virtual/) is now available: the 3 km CONUS model, 48-hour forecasts, with the **full set of vertical levels** — 39 isobaric and 50 hybrid model levels — alongside the surface fields. `[confirm final coverage/levels + catalog URL before publishing]`

What's new isn't only the data, it's how it's served. HRRR is our first **virtual** dataset. Instead of re-copying terabytes into our own store, each chunk points straight at NOAA's public GRIB archive and is decoded the instant you read it. You open it exactly like every other dataset in the catalog — it just doesn't cost a giant duplication to exist. Same three lines:

```python
import dynamical_catalog as dc
ds = dc.open("noaa-hrrr-forecast-48-hour-virtual")
```

That "virtual" idea is the thread running through most of the last six weeks.

## The plumbing behind "virtual"

Making a dataset you can read without copying it took a real overhaul of how we write data. We split the write path into materialized and virtual halves, built a virtual write loop that stores chunk *references* into an Icechunk store and commits them atomically, taught the pipeline about **vertical-level groups** (so pressure and model levels come along), and added CF flag metadata for categorical fields. We shook it all out on a staging catalog first — GEFS 10-day spatial and HRRR spatial rode it before today — so nothing hit the live catalog until it was proven. **GEFS 10-day spatial is right behind HRRR** on the same rails. `[MM: add why this matters to you / what's next]`

## status.dynamical.org is live

We stood up [**status.dynamical.org**](https://status.dynamical.org) — a public view of whether each dataset updated on time, backed by real per-dataset SLA budgets. It grew webhook subscriptions (get pinged when your dataset updates, including Slack messages with forecast charts) and a management UI on top. If you build on our data operationally, this is how you'll know it's fresh. `[confirm what's ready for external use]`

## Validation reports, deeper

The [per-dataset validation reports](https://dynamical.org/catalog/) we started in June got richer — full-period value time-series plots joined the null-fraction traces and spatial comparisons, so you can sanity-check a variable across the whole archive before you trust it.

## A cleaner catalog and site

- Catalog pages now document **chunk and shard sizes** per dataset — element counts, coordinate spans, and uncompressed sizes — so you can align your reads to the grid.
- Dataset pages split into tidy **Forecasts** and **Analyses** tables, with vertical-level variables grouped instead of scrolling forever.
- We reorganized the site around research **areas** and **projects**, refreshed the homepage, and made the whole thing legible to search engines and LLMs (`llms.txt`, structured metadata, sitemap).

## The library: dynamical-catalog v0.7

[`dynamical-catalog`](https://pypi.org/project/dynamical-catalog/) reached **v0.7.0** — newer Zarr (3.2) and Icechunk, plus the `gribberish` GRIB decoder that makes the virtual datasets above actually readable. `pip install -U dynamical-catalog`.
